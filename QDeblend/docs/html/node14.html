The File Menu

Load Cube:
Load a 3D datacube in FITS format with spatial dimension in the and axis and spectral dimension in the -axis. The data must be stored in the first extension if the file contains multiple extensions. The wavelength information must be stored in the primary FITS header as keywords CRVAL3 (the wavelength of the first monochromatic image) and CDELT3 (linear wavelength dispersion per pixel). The wavelengthes should be in units of Å.


Save Cubes:
This item is only available after the deblending process was finished successfully. In the following dialog the user is requested to select a directory and to provide a PREFIX for the file names. The datacubes of the deblended QSO and host emission will then be stored in the given directory as PREFIX_QSO.fits and PREFIX_host.fits. The wavelength range for these two files can optionally be reduced to cover only a specific spectral region.
Figure 4: Save Cubes dialog

Save Session:
All the parameters and settings from the Deblending Control Widget can be stored in a special (q3d) binary file, so that the deblending can be repeated later or paremeters can easily be adjusted without starting from the beginning.
Load Session:
Load a q3d file to recover a previous session for a specific datacube.
